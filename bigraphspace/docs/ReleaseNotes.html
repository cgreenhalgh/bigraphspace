Release Notes - ReactiveXML/Bigraphs

Chris Greenhalgh

Known issues

  • The abstraction for Match is massively incomplete, and there is no abstract access to Matcher/Transformer (c.f. Bigraph vs DomBigraph)
  • There is currently no support in the Definitions XML or the code to specify that a rule applies only if the context of the match is (transitively) active. 
  • BigraphFinder has only a single  global default - should allow (e.g. class-loader specific default for use in simulator)
  • DomReactiveBigraph has some placeholder fields but doesn't actually support creation as a sub-bigraph.
  • RuleCondition evaluation does not do active test.

Changes

  • Starting point of abstracted Match interface.
2009-06-25
